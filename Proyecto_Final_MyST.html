1.0 Introducción¶

El trading requiere mucha atención y sensibilidad al mercado. Los traders experimentados confían en múltiples fuentes de información, como noticias, datos históricos, informes de ganancias e información privilegiada de la empresa. El riesgo es alto y es necesario considerar muchas variables. Por esa razón, algunas instituciones financieras dependen exclusivamente de las máquinas para realizar transacciones. Eso significa que una computadora con conexiones a internet de alta velocidad puede ejecutar miles de operaciones durante un día obteniendo ganancias con una pequeña diferencia en los precios. A esto se le llama trading de alta frecuencia. El trading intradía es muy riesgoso debido al comportamiento a corto plazo de los mercados que reflejan miles de millones de valores que fluctúan rápidamente en respuesta a condiciones cambiantes que se aproximan a una caminata aleatoria.

1.1 Redes Neuronales¶

Cuando los operadores utilizan datos históricos junto con indicadores técnicos para predecir el movimiento de las acciones, buscan patrones familiares. Por ello algunos tipos de redes neuronales son excelentes para este tipo de problemas. Una red neuronal puede reconocer patrones a partir de datos históricos, tales como el precio de las acciones un determinado horizonte de tiempo.

2.0 Objetivos¶

La intención del presente trabajo es proponer, crear y analizar un sistema de trading que utiliza machine learning para la generación señales de compra/venta del BTC/USD operado en MetaTrade5. Así como evaluar su desempeño y efectividad a lo largo de un determinado horizonte de tiempo por medio de un backtest.

Se propuso utilizar una red nuronal multicapa para reconocer patrones en datos de precios históricos, ya que estas puden tomar cualquier cantidad de funciones y aprender de ellas simultáneamente.

En este proyecto, la red tuvo que aprender de secuencias de una hora y predecir la diferencia en el precio para el siguiente periodo. Es decir, la diferencia entre el Open y Close. Si la diferencia fue positiva, se genera una señal de compra para el periodo inmediato siguiente, en el caso contrario, se genera una señal de venta.

Al momento de hacer trading existen cuatro elementos o consideraciones básicas para llevar un seguimiento más efectivo y organizado.

Tales como:

  1. Uso de datos

    BTC/USD 1H OHLCV

  2. Generación de señales

    Red neuronal multicapa | Etiqueta: diferencia en el precio entre el precio de apertura y cierre del periodo. Si este es positivo, se ejecuta una compra. Si es negativo se abre una posición de venta.

  3. Dimensionamiento de la posiciones

    El volumen de contratos operados en cada transacción se mantiene fijo a 1 unidad.

  4. Toma de pérdidas y ganancias

    Cada vez que se genere una señal de compra/venta automaticamente se toman pérdidas o ganancia de la que se encuentre actualmente abierta y se abre simultaneamente la nueva.

9.0 Conclusiones y comentarios finales¶

  • Aunque pudiese parecer que los resultados del modelo no eran muy prometedores, el conjunto de las señales con los criterios propuestos de volumen, tiempo y toma de ganancia resultó en una excelente combinación.

  • Atribuimos el buen desempeño del sistema de trading al hecho de que a pesar de que el modelo pronosticaba una diferencia entre el precio de apertura y cierre para el siguiente periodo, la decisión para operar fue solamente tomar el signo, es decir la dirección del precio del activo.

  • Dado que el volumen se mantuvo constante (1 contrato) y tambien el tiempo entre cada operación abierta no fue más de una hora, se pudo limitar hasta cierto punto la exposición al riesgo. En donde la máxima pérdida flotante fue de tan solo -$194 dólares, lo cual no es ni el 1.0% del capital.

  • Se puede observar en la gráfica superior que el comportamiento del BTC/USD fue bastante errático, ya que después de alcanzar un pico de 54.5K este se desplomó hasta los 52.7K.

  • El modelo logró adaptarse lo suficiente para arrojar la señales correctas e incrementar positivamente el capital +3.5K dólares por arriba de cuando se inició. Esto es un rendimiento efectivo del 3.5% en menos de un día.

  • Una posible explicación del desempeño tan sobresaliente es que de los 4 meses de precios uilizados, el set de datos de validación fue de solamente 1 día. Para verificar si el resultado obtenido fue simplemente una cuestion de azar, se repitió el procedimiento de: entrenamiento, validación y backtest con diversos instrumentos y periodos distintos para cada uno. Sorprendentemente, el Sistema de Trading tuvo un comportamiento y resultados similares a los mostrados en el presente notebook. Para lograr que el sistema de trading diera malos resultados, se tuvo que cambiar el tamaño del conjunto de datos de validación. Es decir, si el dataset original se dividía por ejemplo: 80% de datos para entrenamiento, y 20% de datos para validación, la estrategia era perdedora en casi todos los casos.

  • Lo anterior, traducido a la práctica, significaría que cada día (al inicio del día por ejemplo) el modelo se reentrenaría con 3 o 4 meses de precios y se "echaría a andar" el resto del día. Al día siguiente, el modelo se reentrena y así sucesivamente.

  • Por las pruebas, experimentos realizados y los resultados obtenidos, sospechamos que el querer generar un modelo (one model to rule them all) que sea efectivo y perdure a lo largo del tiempo, es una tarea muy difícil (quizá imposible). Si el enfoque en la práctica del trading se cambia y se generan nuevos modelos frecuentemente, que sean efectivos para el corto plazo, entonces, tal vez, sí sea posible generar un sistema de trading altamente efectivo en términos monetarios.
